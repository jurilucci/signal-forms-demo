Submit e reset

Come funziona

  • submit(form, async (form) => { ... }) gestisce il flusso: marca tutti i campi come touched, verifica valid(); se non valido non chiama l'action. Durante l'esecuzione form().submitting() è true.
  • Puoi ritornare errori server: return [{ field: myForm.email, error: { kind: 'server', message: '...' } }] e vengono applicati ai campi.
  • form().reset() resetta lo stato di interazione (touched, dirty). Opzionalmente puoi passare un nuovo valore: reset({ email: '', ... }).
