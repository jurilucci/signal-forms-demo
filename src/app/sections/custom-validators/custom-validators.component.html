3. Validatori custom

Come funziona

Con validate(path, (ctx) => ...) puoi definire validatori custom. Il contesto ctx espone:

  • value() – valore del campo corrente
  • valueOf(path) – valore di un altro campo (es. valueOf(f.password))
  • state / stateOf(path) – stato (touched, dirty, ecc.)

Ritorni undefined se ok, oppure un oggetto { kind: string, message?: string } (o un array) per segnalare errore.

Reattività: il validatore viene rieseguito quando cambia qualsiasi signal che leggi (value(), valueOf(...)). Es: cambiando password, il validatore di confirmPassword si ricalcola da solo, senza updateValueAndValidity().
